## Spacing Scale

Declared values (must be multiples of 4), matching the rem-based spacing already used throughout `public/form.css` / `public/button.css` / `pages/notifications.tsx` (1rem = 16px base):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Gap between an emoji glyph and adjacent text (e.g. `ЁЯСе` + group name) |
| sm | 8px | Gap between avatar and name within a group row; gap inside `.notification-actions`-style action clusters |
| md | 16px | Default internal padding/margins; gap between form fields; `.form-group` help-text spacing |
| lg | 24px | Group-row padding (matches existing `.notification-item` `padding: 1.5rem`); `.form-group` `margin-bottom: 1.5rem` |
| xl | 32px | Vertical gap between major sections on `/groups` (existing `.content` pattern) |
| 2xl | 48px | Not used by this phase's new UI |
| 3xl | 64px | Not used by this phase's new UI |

Exceptions:
- **Mode `<select>` touch target:** override the default `select` padding from `public/form.css` (`0.75rem` тЙИ 34тАУ38px total height) to an explicit `min-height: 44px` on the per-group mode dropdown, since `/groups` is used from `pages/mobile.tsx` contexts and this is the phase's primary tap target. Applies only to `.group-row select[data-bind^="mode_"]`.
- **Group avatar image:** 40px ├Ч 40px circular (`border-radius: 50%`), a component asset size, not a spacing token тАФ no existing precedent in the codebase to reuse, chosen to sit visually between the 24px `stat-number`-adjacent icons and the existing `.notification-item` 24px row padding.

## Typography

Reusing the sizes/weights already present in `public/form.css`, `public/button.css`, and `pages/notifications.tsx`'s `notificationStyles` (converted from rem at 16px base). Exactly 2 weights are used by this phase's **new** UI (regular 400, semibold 600) тАФ the existing page `<h1>` (28px, browser-default bold from `Layout.tsx`, unchanged by this phase) is inherited, not redeclared.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 |
| Label | 14px | 600 | 1.4 |
| Heading (group row name / card title) | 18px | 600 | 1.3 |
| Display (page `<h1>`, inherited from `Layout.tsx`, unchanged) | 28px | 700 (existing UA-bold, out of scope to change) | 1.2 |

Notes:
- `Body` (16px/400) = group help text, dropdown option text, default paragraph copy тАФ matches existing `input`/`textarea` `font-size: 1rem` in `public/form.css`.
- `Label` (14px/600) = per-group mode dropdown's implicit label context and the `/notifications` Groups card summary line тАФ matches `.notification-description` (`0.9rem` тЙИ 14px) combined with `label { font-weight: 600 }` from `public/form.css`.
- `Heading` (18px/600) = each group row's name text and the Groups card title тАФ matches `.notification-name` (`1.1rem` тЙИ 18px, `font-weight: 600`) already established in `pages/notifications.tsx`.

## Color

Existing brand palette across `public/layout.css` / `public/button.css` / `pages/notifications.tsx` is a purple/indigo (`#667eea` тЖТ `#764ba2`) accent on a white content surface with a dark slate header. This phase reuses it exactly тАФ no new brand colors are introduced.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` | `.container` / `.content` background тАФ the page body every group row and form field sits on |
| Secondary (30%) | `#2d3748` (header) and the `#667eea тЖТ #764ba2` body gradient backdrop | `.header` bar (existing, unchanged) and the page backdrop visible around `.container`; also the neutral grays `#4a5568` / `#718096` used for row/help text |
| Accent (10%) | `#667eea` | Reserved for: the existing `Save Group Settings` primary button (`.btn-primary`, unchanged); the mode `<select>` focus ring/border (inherited automatically from `public/form.css` `select:focus`); the "All messages" count in the `/notifications` Groups summary line, to draw the eye to the most-active state |
| Destructive | `#f56565` | Existing `.btn-danger` / `.error-message` token тАФ **not used by this phase's new UI** (no destructive actions in scope; kept declared for consistency with the rest of the app's error banner, which the PATCH error path on `/groups` already reuses unchanged) |

Accent reserved for (explicit тАФ nothing else in this phase's new UI may use `#667eea`): the unchanged `Save Group Settings` button, the mode dropdown's native focus ring, and the "All" count number in the Groups summary card.

### Per-group mode status colors (new, this phase)

The existing app already has a two-color status-badge convention (`.notification-status.enabled` / `.disabled` in `pages/notifications.tsx`) for boolean on/off state. This phase introduces a **third** state (`mentions`) between "fully on" and "off," so a third badge color is required. Follow the same light-background/dark-text/border pairing convention already established, extended with a warning-amber tone (standard, high-contrast pairing consistent with the existing green/red pair):

| Mode | Background | Text | Border | Rationale |
|------|-----------|------|--------|-----------|
| All messages | `#d4edda` | `#155724` | `#c3e6cb` | Reuses the existing `.notification-status.enabled` triple exactly тАФ "All" is the most-active state, same visual weight as "Enabled" elsewhere in the app |
| Only @mentions | `#fff3cd` | `#856404` | `#ffeaa7` | New тАФ amber/warning tone signals "partial/conditional," sitting visually between the green "all" and red "muted" states, without introducing a fourth unrelated hue |
| Muted | `#f8d7da` | `#721c24` | `#f5c6cb` | Reuses the existing `.notification-status.disabled` triple exactly тАФ "Muted" reads as the same visual weight as "Disabled" elsewhere in the app |

These three pairs are used both for each group row's mode badge/indicator (if the mode dropdown is paired with a small status dot/badge) and for the `/notifications` Groups card's per-mode counts (D-05).

| Row-level fallback (Pitfall 3: metadata fetch times out) | Name falls back to **Unnamed group** in muted gray (`#718096`, matching `.help-text`); the row still renders with a placeholder avatar (initials or a generic ЁЯСе glyph in a 40px circle) and its mode dropdown remains fully functional тАФ a slow/missing kind 39000 metadata fetch must never block the ability to set a group's mode |
| Error state (PATCH failure) | Reuses the existing `.error-message` pattern verbatim: "тЭМ {error message}" bound to `$error`, exactly as already implemented in `pages/groups.tsx` тАФ no new error copy needed, the same banner now also covers per-group mode save failures |
