    1. `export type GroupNotificationMode = "all" | "mentions" | "muted";` — this file is the SINGLE source of truth for the type; services/config.ts will `import type` it (do not redeclare it there). Per RESEARCH "Supporting", it is a plain string union, not a schema-library type.

    3. `export function messageMentionsPubkey(message: NostrEvent, pubkey: string): boolean` — returns true if any tag is a `p` tag whose value equals `pubkey` (short-circuit), otherwise returns whether any pointer from `getContentPointers(message.content)` resolves via `getPubkeyFromDecodeResult(pointer)` to `pubkey`. This is D-02's OR semantics. Do NOT hand-roll a bech32/nostr: regex — use the applesauce helpers (they handle the optional `nostr:` prefix, boundary matching, and malformed-string safety per RESEARCH "Don't Hand-Roll").

    4. `export function passesGroupModeGate(mode: GroupNotificationMode, message: NostrEvent, userPubkey: string): boolean` — exhaustive switch: `muted` returns false, `mentions` returns messageMentionsPubkey(message, userPubkey), `all` returns true (D-01/D-09 step 2). Rely on the exhaustive union so strict mode confirms all cases are handled.

    5. `export function getGroupMode(modes: Record<string, GroupNotificationMode>, group: GroupPointer): GroupNotificationMode` — returns `modes[encodeGroupPointer(group)] ?? DEFAULT_GROUP_NOTIFICATION_MODE` (D-06 fallback).

    6. `export function summarizeGroupModes(modes: Record<string, GroupNotificationMode>): { all: number; mentions: number; muted: number }` — initialize counts to zero and increment per value in Object.values(modes) (D-05).

    7. `export function isGroupNotificationMode(value: unknown): value is GroupNotificationMode` — returns true only when value is exactly "all", "mentions", or "muted". This is the reusable ASVS V5 input validator the /groups PATCH handler (Plan 04) will call on each submitted mode signal so arbitrary strings never reach the config map.
