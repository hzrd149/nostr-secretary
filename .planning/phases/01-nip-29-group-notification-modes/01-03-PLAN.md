<objective>
Wire the per-group mode decision into the live NIP-29 notification pipeline so
that an incoming group message is evaluated as: (1) master switch, (2) per-group
mode + @mention gate, (3) existing sender gate, (4) send. Steps 1, 3, and 4
already exist and stay byte-for-byte unchanged; this plan inserts only step 2 —
the pure functions built in Plan 01 — at the top of the existing `.subscribe()`
callback. It also lands the D-09 truth-table test.

Purpose: This is where D-01/D-06/D-08/D-09 become real behavior. The layering
order is load-bearing: the mode gate must sit BEFORE `shouldNotify` and must never
be folded into it (Pitfall 5) — `shouldNotify` has three sibling copies in other
notification modules and any shared refactor is out of scope.

Output: `notifications/groups.ts` with the mode gate inserted, and
`tests/notifications/groups.test.ts` proving the full D-09 truth table via the
exported pure helpers.
</objective>
