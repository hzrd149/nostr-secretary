<objective>
Build the per-group notification controls on the existing /groups page: a list of
the user's joined NIP-29 groups, each row showing the group's picture and name and
a mode dropdown (All messages / Only @mentions / Muted) preset to that group's
current effective mode, plus an extended save (PATCH) that validates and persists
each mode into config.groups.modes. The page keeps its existing global controls
(enable toggle, group-link template, whitelist/blacklist) unchanged.

Purpose: This is the D-03/D-04 user-facing surface and the phase's one mutating
HTTP surface. Two landmines dominate: Datastar signal-name safety (index-based
signal names, re-keyed server-side — Pitfall 2) and the same-named getGroupMetadata
collision (alias on import — Pitfall 4). It also carries the ASVS V5 input-validation
control (T-01-01): the PATCH must never write an unrecognized mode string into the
config map.

Output: `pages/groups.tsx` with an async GET that renders the group list and a PATCH
that positionally zips validated mode signals into config.groups.modes.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser/client → /groups PATCH | Untrusted `mode_N` Datastar signals submitted by the client cross into server-side config mutation |
| relay → /groups GET | Untrusted kind-39000 group metadata (picture/name) is fetched and rendered |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-01 | Tampering | Arbitrary string injected into `groups.modes` via a `mode_N` signal | high | mitigate | Validate every submitted `mode_N` with `isGroupNotificationMode` before writing to `config$`; ignore any non-enum value (ASVS V5). Proven by the validator's unit tests in Plan 01 |
| T-01-02 | Cross-Site Scripting | Untrusted group name/picture from kind-39000 metadata rendered into the row | medium | mitigate | Render the group name with the `safe` attribute (@kitajs/html auto-escape); picture is used only as an `src` attribute, not as markup |
| T-01-P1 | Spoofing / Elevation | No authentication on the /groups route | high | accept | Pre-existing, project-wide gap documented in `.planning/codebase/CONCERNS.md`; explicitly out of scope per RESEARCH Security Domain. The new PATCH surface is no more exposed than the existing whitelist/blacklist PATCH on the same page |
| T-01-P2 | Tampering (CSRF) | No CSRF token on mutating routes | high | accept | Pre-existing, project-wide gap (`.planning/codebase/CONCERNS.md`); out of scope for this phase; not introduced by this plan |
