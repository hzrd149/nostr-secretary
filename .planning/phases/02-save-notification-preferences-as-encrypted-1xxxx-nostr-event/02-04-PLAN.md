<objective>
Close the D2-12 user-facing loop: on the notification settings overview, surface a
non-blocking hint that read-only (no-signer) users can connect a signer to sync
their settings across devices. This is the only new UI in the phase — everything
else is behind the scenes. The hint reads the same enabled$ signal the sync
listener uses, so what the user sees matches whether sync is actually active.

Read-only users are explicitly unaffected: the page continues to render and all
local settings continue to work; the hint is additive and never blocks.

Purpose: Without this, a no-signer user has no indication that connecting a signer
unlocks cross-device sync — the feature would be invisible to exactly the users it
would help most.

Output: a small hint element in pages/notifications.tsx, gated on signer presence.
</objective>
