<objective>
Fix the single concrete bug behind this whole phase (D3-02): NIP-46 bunkers only
honor the permissions they were granted at connect time, and SIGNER_PERMISSIONS
in const.ts requests NIP-44 encrypt/decrypt but never nip04_decrypt — so every
bunker connection made through this app is silently unable to decrypt legacy
kind-4 DMs (they route through signer.nip04.decrypt, which the bunker rejects).
Add "nip04_decrypt" to the requested permissions, using the applesauce-provided
Permission.Nip04Decrypt constant, and lock it in with tests. Deliberately do NOT
add nip04_encrypt: this is a receive-only notification secretary with no DM send
surface (D3-03).

This is a review-and-harden change, not a rebuild (D3-01): the decrypt call site
(unlockLegacyMessage) is already correct; only the permission grant is missing.

Purpose: Without this, a user connects a signer, enables DM notifications, and
never gets notified for legacy DMs — the bunker refuses every decrypt request.
Output: const.ts with nip04_decrypt requested; tests/const.test.ts asserting it.
</objective>
