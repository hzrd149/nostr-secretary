<objective>
Add the two default per-context limits to configuration and NIP-78 sync (D7-05/D7-06): top-level
`perGroup` (default 3) and `perDm` (default 5) scalar siblings of `perType` on
`AppConfig.rateLimit`, with matching migration backfill; the same two fields on `SyncedPrefs.rateLimit`
with field-by-field serialization and per-field `asNonNegativeInt` coercion in `asRateLimit`
(absent → local defaults, never 0); a `PREFS_VERSION` bump to 4; and a CHANGELOG entry documenting
the additive behavior change and the `0 = unlimited` escape hatch. Reuse the EXACT existing
`global`/`perType.*` migration + sync patterns verbatim (D7-09) — no new clamp constant (0 is a safe
"unlimited" sentinel for these fields, unlike `window`).

Purpose: Make the default per-group/per-DM limits configurable, durable across restarts, and synced
across devices/clients through the existing encrypted kind-30078 event.
Output: Extended config + preferences modules, regression tests, and a CHANGELOG entry.
</objective>
