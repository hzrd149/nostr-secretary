<objective>
Extend Phase 6's pure, clock-injectable accounting core (`services/rate-limit-accounting.ts`)
with a per-context bucket layer: a flat `contexts: Record<string, number>` map on
`RateLimitState`, two new scalar config fields `perGroup`/`perDm` on `RateLimitConfig`, and a 5th
optional `context?: string` parameter on `evaluate()` that adds a third `underContext` gate
(most-restrictive-wins, D7-04). The per-context map is pruned for free by the EXISTING
tumbling-window rollover (D7-02) — no new TTL/LRU/timer code. Per-context overflow rolls into the
existing per-type `overflow` counts only (D7-07). This is a purely additive extension of the
already-implemented Phase-6 module (D7-09): with no `context` argument, `evaluate()` must behave
byte-identical to today.

Purpose: Deliver the runtime "default limit on join" — a new group/DM automatically gets its own
default-limited bucket the first time a notification for it is evaluated, with zero persisted
per-context config (D7-02).
Output: The extended accounting module + a comprehensive injected-clock test matrix.
</objective>
