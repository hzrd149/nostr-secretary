<objective>
Wire the Wave-1 foundations into the live pipeline: layer a per-category enable gate into
BOTH DM listeners in notifications/messages.ts (NIP-04 and NIP-17), completing the phase's
user-visible behavior. For each incoming DM, classify the real sender as contacts/others via
classifyDmSender(await isContact(sender)) and stop if that category's enabled flag is off --
inserted as a NEW, separate statement immediately BEFORE the existing shouldNotify check, so
the mute + whitelist/blacklist layer (shouldNotify) stays byte-identical and still applies on
top when the category is on (D5-07 layering, mirroring Phase-1 D-09). The classification
sender is the DM's real author (`sender` for NIP-04, `rumor.pubkey` for NIP-17), never the
gift wrap's random pubkey (D5-09). Because isContact falls back to false on an unresolved
follow list, an unknown sender classifies as others and the listener never crashes (D5-02).

Coverage is a new WR-04-style truth-table mirror in tests/notifications/messages.test.ts
that imports the real (pure) classifyDmSender and mirrors the messages[category].enabled
lookup -- category-off stops before shouldNotify; category-on falls through to the unchanged
shouldNotify mirror -- exactly how shouldNotify's own gate order is tested today, without
importing the self-subscribing modules.

Purpose: deliver the split-DM behavior on top of Plan 01's classifier/isContact and Plan 02's
config schema, without touching the decrypt paths or the existing sender gate. Output: the
category gate in both listeners + its truth-table test; `bun run lint` and full `bun test` green.
</objective>
